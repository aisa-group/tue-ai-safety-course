What Are Adversarial Examples?

Definition: Given a classifier $f$ and input $x$ with true label $y$, an adversarial example $x' = x + \delta$ satisfies:
$$f(x') \neq y \quad \text{s.t.} \quad \|\delta\|_p \leq \epsilon$$

Common $L_p$ Norms

$\|\delta\|_0 = \#\{i : \delta_i \neq 0\}$ — sparse pixels

$\|\delta\|_2 = \sqrt{\sum_i \delta_i^2}$ — Euclidean distance

$\|\delta\|_\infty = \max_i |\delta_i|$ — max per-pixel change

Projected Gradient Descent (PGD)

$x^{t+1} = \Pi_{x+\mathcal{S}}\!\Big(x^t + \alpha\,\text{sign}\big(\nabla_{x^t} J(\theta, x^t, y)\big)\Big)$
  • Multi-step generalization of FGSM
  • Random start within $\epsilon$-ball
  • Project back after each step
  • "First-order adversary" — strongest using only gradients

Carlini & Wagner (C&W) Attack

$\min_\delta \|\delta\|_2 + c \cdot f(x + \delta)$

where $f(x') = \max\!\big(\max_{i \neq t} Z(x')_i - Z(x')_t,\; -\kappa\big)$
  • Optimization-based: directly minimizes distortion
  • Uses logit-layer objective (avoids gradient masking)
  • Change of variables: $\delta = \frac{1}{2}(\tanh(w)+1) - x$ ensures valid pixel range
Impact: 100% success rate on all tested defenses. Completely broke defensive distillation (reducing claimed 99.5% robustness to 0%).

Defenses & The Arms Race

Adversarial Training, Certification & Scaling

Adversarial Training: The Gold Standard

PGD Adversarial Training

$$\min_\theta \mathbb{E}_{(x,y)\sim\mathcal{D}}\!\left[\max_{\|\delta\|\leq\epsilon}\mathcal{L}(f_\theta(x+\delta),y)\right]$$

Min-max optimization: inner maximization finds worst-case perturbation, outer minimization trains the model to be robust against it.

TRADES

$$\min_\theta \mathbb{E}\!\left[\mathcal{L}(f_\theta(x),y) + \beta \cdot \text{KL}(f_\theta(x)\|f_\theta(x'))\right]$$

Decomposes robustness into natural accuracy + boundary smoothness. β controls the tradeoff.

Madry et al., ICLR 2018 · Zhang et al. (TRADES), ICML 2019

Adversarial training smooths the loss landscape, making gradients more interpretable

The Robustness–Accuracy Tradeoff

Tsipras et al., ICLR 2019 · Schmidt et al., NeurIPS 2018

Theorem (Tsipras et al.):
Any classifier with ≥(1−δ) standard accuracy
has robust accuracy ≤ p/(1−p)·δ
The tradeoff is fundamental, not an artifact of weak defenses. Robust classifiers must ignore non-robust features that boost clean accuracy.

Defending LLMs Against Jailbreaks

SmoothLLM

Randomly perturb copies of input, aggregate predictions. Reduces GCG: 98% → <1%. Robey et al., 2023

Perplexity Filtering

Detect nonsensical suffixes. Effective vs GCG (PPL ~1532) but fails vs PAIR (natural language).

Representation Engineering

Read & control model activations to enforce safety. TruthfulQA: +18pp via honesty directions. Zou et al., 2023

No single defense is universal. Layered defense: input-level (perplexity) + model-level (representation engineering) + output-level (safety classifier).

Scaling Laws for Adversarial Robustness

Bartoldson et al., ICML 2024

$$\text{RobAcc}(N, D) = A - \frac{B}{N^\alpha} - \frac{C}{D^\beta}$$
Adversarial robustness follows power-law scaling in model size (N) and data (D). Predicted asymptotic ceiling: ~90%.
~10% of AutoAttack “adversarial examples” are actually invalid — humans disagree with the ground truth label. The ceiling matches human performance.
