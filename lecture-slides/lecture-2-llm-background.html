PART I

The Anatomy of a Transformer

Tokens, Embeddings, Attention, MLPs, MoE & Unembedding

The Decoder-Only Transformer

Figure: Cameron R. Wolfe, “Decoder-Only Transformers” (2023)

  • Tokenizer: text → integer IDs
  • Embedding: ID → $d$-dimensional vector
  • Attention: mixes information between positions
  • MLP: per-position computation (features, facts)
  • Residual stream: the additive “backbone”; every block reads from and writes to it
  • Unembedding: hidden state → vocabulary probabilities
  • Sampling: probability → next token choice

Elhage et al., “A Mathematical Framework for Transformer Circuits,” 2021

Byte-Pair Encoding (BPE)

The Algorithm

  1. Start with all individual bytes (256 tokens)
  2. Count adjacent pair frequencies in the corpus
  3. Merge the most frequent pair into a new token
  4. Repeat until desired vocabulary size
Corpus: "aaabdaaabac"
Merge 1: aa → Z  →  "ZabdZabac"
Merge 2: Za → Y  →  "YbdYbac"
Merge 3: Yb → X  →  "XdXac"

The Model Sees Integer Sequences

After tokenization, the model operates on sequences of integer IDs:

“strawberry” → GPT-4: [str 496aw 675berry 15717]
“strawberry” → GPT-4o: [st 302raw 1618berry 19772]
“Tübingen” → GPT-4: [T 51ü 2448bing 7278en 268]
“Tübingen” → GPT-4o: [T 51üb 60264ingen 5237]
“hello world” → GPT-4: [hello 15339 world 1917]

Consequences of Tokenization

The Strawberry Problem

“How many r’s in strawberry?” — LLMs consistently answer 2 (correct: 3).

The model sees [496, 675, 15717] (“str”, “aw”, “berry”), never individual characters.

Generalizes to all character-level tasks: counting letters, anagrams, spelling backwards, detecting double letters.

Numerical perception

BPE shards numbers inconsistently: 12345 may tokenize as [1234][5] or [12][345]. The model sees a jumble of multi-digit fragments rather than a single numeric quantity.

Multilingual blind spots

BPE is biased toward the tokenizer-training corpus. Low-resource languages are compressed less efficiently, e.g. 5–10× more tokens per word in Thai than in English.

Embedding: Discrete → Continuous

Every token ID i is mapped to a row of a learned matrix:

$\text{embed}(i) = E_{i,\,:} \in \mathbb{R}^{d}, \quad E \in \mathbb{R}^{V \times d}$

$V$ is the vocabulary size (e.g. 128K for Llama 3), $d$ is the hidden dim (e.g. 4096 for Llama 3 8B).

The embedding table is simply a lookup: the only place the integer index matters.

Glitch tokens are semantic collisions

The vocabulary is built from tokenizer-training data. The embedding matrix $E$ is trained on the (different) model-training data.

Rumbelow & Watkins, “SolidGoldMagikarp,” 2023

Attention — Mixing in the Sequence Dimension

$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$

For one head, per token position: $Q,K \in \mathbb{R}^{T\times d_k}$, $V \in \mathbb{R}^{T\times d_v}$, where $T$ is the sequence length and $d_k = d_\text{model}/h$ splits the hidden dimension across $h$ heads.

$QK^\top \in \mathbb{R}^{T\times T}$: a token×token similarity matrix.

Multi-head: $h$ parallel heads, each learning a different mixing pattern across positions.

Causal mask: each position only attends to earlier positions.

Head 8-10 — attends from verbs to their direct objects.

Head 4-10 — attends to passive auxiliaries.

Head 5-4 — attends from pronouns to their antecedents.

Clark et al., “What Does BERT Look At?” 2019

MLPs — Mixing in the Hidden Dimension

Structure

SwiGLU (Llama, DeepSeek, Qwen):

$\text{MLP}(x) = W_2 \bigl(\text{silu}(W_1 x) \odot W_3 x\bigr)$

Three weight matrices: gate ($W_1$), up ($W_3$), down ($W_2$).

Operates independently at each position, no mixing between tokens.

Intermediate dim typically $\frac{8}{3}d$ or larger, where most parameters live.

MLP at one position: expand to a wide intermediate space, gate, project back.

Mixture of Experts (MoEs) — Sparse Mixing

Replace each dense MLP with $N$ expert MLPs + a learned router.

Router: $\text{TopK}(\text{softmax}(W_r \cdot x))$ picks which $k$ experts process each token. Only $k$ fire → total params >> active params.

Fedus et al., “Switch Transformers,” JMLR 2022

How Information Flows

Two information highways

Residual stream (vertical): state vector persists across layers at each token position.

Attention (horizontal): information flows between positions at each layer via K/V projections.

Each layer adds to the stream. Additive structure → representations are approximately linear in the residual stream.

Elhage et al., “A Mathematical Framework for Transformer Circuits,” 2021

Unembedding & Sampling

Unembedding

$\mathbf{h} \mapsto W_U \mathbf{h} \in \mathbb{R}^V \xrightarrow{\text{softmax}} p$

Linear projection from hidden state to vocabulary-sized logits, then softmax over all tokens. Often $W_U = W_E^\top$ (tied weights).

At every step, the model produces a full distribution $p$. Sampling selects which token to emit.

Sampling Strategies

StrategyMethod
GreedyPick $\arg\max_i p_i$
TemperatureScale logits by $1/T$ before softmax
Top-kKeep the $k$ most probable tokens
Top-p (nucleus)Keep smallest set with cumulative $\sum p \geq p_{\text{top}}$
Min-pKeep $p_i \geq p_{\text{base}} \cdot \max_j p_j$  (after temperature; $p_{\text{base}}$ ~ 0.05–0.1)

Nguyen et al., “Min-p Sampling,” ICLR 2025 oral

Training Compute: $C \approx 6ND$

$C \approx 6 \cdot N \cdot D$

$C$ = FLOPs, $N$ = parameters, $D$ = tokens.

The 6 is three matmul passes, each costing 2 FLOPs/param/token:

  • Forward: 2
  • Backward w.r.t. activations (propagating the signal): 2
  • Backward w.r.t. weights (computing $\partial\mathcal{L}/\partial W$): 2

For MoE: use active params, not total.

Concrete Examples

70B params × 16 bytes = 1.12 TB

One H100 has 80 GB → need ≥14 GPUs just for the model state.

405B params × 16 bytes = 6.48 TB → ≥81 GPUs.

Activation memory (the forward pass, checkpointed or not) is on top of this and scales with batch and sequence length.

Parallelism: TP, PP, DP

TP — Tensor Parallelism

Shard each layer’s matrices across GPUs within a node.

“Every GPU computes part of every layer”

PP — Pipeline Parallelism

Assign different layers to different GPUs across nodes.

“Each GPU computes a few full layers”

DP — Data Parallelism

Replicate the model, split data batches across replicas, all-reduce gradients.

“Many copies see different data”

Typical hierarchy: TP within node (8 GPUs) → PP across ~16 nodes → DP across the rest of the cluster.

Training Curriculum

Training is a sequence of stages.

  1. Foundation: broad web, code, math.
  2. Specialization: curated domain data, higher quality.
  3. Annealing: LR decay on the best subset.
  4. Post-training: SFT, preference, RL with verifiers.

RLHF

HuggingFace, “Illustrating RLHF”

  1. SFT on demonstrations (~10K–100K examples)
  2. Reward model trained on human comparisons (A > B)
  3. PPO optimizes against the reward model
$\max_\theta \mathbb{E}[R(x,y)] - \beta \cdot \text{KL}(\pi_\theta \| \pi_{\text{ref}})$

Human preferences enter only through the reward model.

Reinforcement Learning with Verifiable Feedback

Replace human preferences with programmatic verification: math answers, passing unit tests, format checks.

$A_i = \dfrac{r_i - \text{mean}(\{r_1,\ldots,r_G\})}{\text{std}(\{r_1,\ldots,r_G\})}$

GRPO: sample a group of $G$ responses, score each with the verifier, use the group-relative z-score as advantage. No learned reward model.

DeepSeek-R1

Trained with RLVR only. Emergent behaviors: spontaneous chain-of-thought, self-verification (“wait, let me check…”), the “aha moment.”

The model learns to “think longer” without being told to.

PART III

How Are Models Used?

Inference, KV Cache, System Prompts & Serving at Scale

Inference: Prefill & Decode

Prefill (Prompt Processing)

Process the whole prompt in parallel. One forward pass, all positions at once. Compute-bound.

Decode (Token Generation)

Generate tokens one at a time. Each step is a forward pass with sequence length 1. Memory-bound.

Why Decode Is Slow

Each new token requires a full forward pass through the model.

Llama 3 70B: ~140 GB of weights must be read per token.

One H100 has 3.35 TB/s memory bandwidth → ≥ 42 ms / token → ~24 tok/sec max.

KV Cache

Without cache: recompute attention over all $t$ tokens each step → $O(t^2)$ decode.

With cache: store K,V from previous tokens, only project the new token → $O(t)$ per step.

Sebastian Raschka, “KV Cache from Scratch”

Llama 3 70B — cache size

80 layers, 8 KV heads (GQA), $d_\text{head}$ = 128, FP16. Per token: ~320 KB.

Serving Many Users: Continuous Batching

Static: all sequences in a batch must finish before new ones start → GPU idles on short sequences.

Continuous: new requests join the batch as old ones finish → ~23× throughput in practice.

PART IV

Model Behavior

Missing Context, Persona & What LLMs “Are”

Where Does “Behavior” Come From?

An LLM is trained only on next-token prediction and preference optimization. It has no module labeled “honesty,” no slot for “persona,” no ledger of “what it knows.”

And yet at deployment we talk about models as if they are honest, sycophantic, self-aware, deceptive, scheming.

The gap: the training objective cares only about next-token likelihood and preference scores. Concepts like truth, self-reference, user intent, and identity are learned implicitly from data.

What Is an LLM Like? Competing Framings

Bender et al., FAccT 2021 · Janus, “Simulators,” 2022 · @TetraspaceWest, shoggoth meme 2022 · nostalgebraist, “the void,” 2025

1. Stochastic Parrot

Statistical pattern matcher with no understanding. “Haphazardly stitching together linguistic forms.”

2. Simulator

A simulation engine that, given context, simulates any situation from the training data; a “multiverse generator” that role-plays any character from training.

3. Shoggoth with Smiley Face

The base model (next-token predictor) is alien/unknowable. RLHF is a thin, friendly mask over the base that is learned late.

4. The Void

“The assistant is defined in a self-referential manner, such that its definition is intrinsically incomplete.” The model is inferring what “assistants are like” from training data and context, and predicts what an assistant would say.

Illustrations by Theia Vogel

Framings → Safety Implications

  • If simulator: jailbreaks work because you’re selecting a different character. The chat template is just another prompt format.
  • If shoggoth: safety alignment is a mask. Recall: safety lives in <1% of params, a single direction, the first few tokens (slide 28).
  • If void: the persona is under-specified. Constitutional AI tries to fill it, but can you specify a character exhaustively?
  • Emergent misalignment (→ Lecture 4): fine-tuning on unrelated tasks can change the persona entirely.

Key Takeaways

I · Architecture

Transformers are stacks of identical blocks reading and writing into a single residual stream. Attention mixes tokens along the sequence dimension; MLPs (and MoE experts) mix along the hidden dimension.

II · Training

Training compute is $6 \cdot N \cdot D$ FLOPs; memory is ~16 bytes/param with Adam. Frontier runs cost \$5M–\$500M and require GW-scale sites. All safety behavior lives in the post-training pipeline.

III · Inference

Prefill is compute-bound; decode is memory-bound. KV cache grows linearly with context. Continuous batching and prefix caching are what make serving affordable. System prompts are just prepended text.

IV · Behavior

Concepts like honesty and persona are not in the loss; they are learned implicitly. The assistant is a fictional character pasted on top of a next-token predictor.
